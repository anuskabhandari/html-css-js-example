
# Responsive Web Design & JavaScript Practice Projects

This repository contains my practice projects in **HTML, CSS, and JavaScript**, focusing on **responsive web design**, **DOM manipulation**, and **interactive web applications**.  

These exercises helped me learn front-end development from **basic to intermediate level**, building small real-world projects like a **To-Do app** and a **Calculator**.

---

##  Projects & Examples

### 1️. DOM Interaction

**Tasks practiced:**

- **Change Text**: Click a button to change heading text.  
- **Change Text & Color**: Heading text changes on click and color changes dynamically.  
- **Show/Hide Element**: Toggle visibility of an element with a button.  
- **Display Name**: Input field and button to display personalized greetings.  
- **Character Counting**: Count the number of characters typed in an input/textarea live.  

**Concepts learned:**  
- DOM selection (`getElementById`, `querySelector`, `querySelectorAll`)  
- `textContent`, `value`, `style` manipulation  
- Event handling (`click`, `input`)  
- Conditional logic and input validation  

---

### 2️. To-Do App

**Features:**

- Add tasks dynamically to a list (`<ul>` + `<li>`).  
- Delete individual tasks or the last task.  
- Input validation to prevent empty tasks.  

**Concepts learned:**  
- `createElement`, `appendChild`, `removeChild`  
- Dynamic updates to the DOM  
- Event-driven programming  

---

### 3️. Calculator App

**Features:**

- Functional buttons for digits `0-9`, operators (`+ - * /`), `=` and `C`.  
- Heading inside the calculator box.  
- Prevent multiple operators in a row.  
- Decimal support.  
- Responsive layout using CSS Grid and Flexbox.  

**Concepts learned:**  
- Query selectors for multiple buttons  
- Event listeners for numbers, operators, equal, and clear  
- Dynamic content update with `eval()`  
- CSS Grid & Flexbox layout, responsive design  

---

### 4. Weather App

**Features:**

Enter any city name to get real-time weather.
Displays temperature (°C) and weather condition.
Clear button to reset input and result.
Responsive, professional design with hover effects.
Handles errors for invalid or empty city input.

**Concepts learned:**

Fetch API to get data from OpenWeatherMap API
DOM manipulation to display results dynamically
Event handling for buttons
Error handling and input validation
CSS styling for professional look

1. Clone or download the repository.

3. Interact with elements (buttons, inputs) to see live updates.
